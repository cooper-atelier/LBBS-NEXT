M3 Plan: Agent Management API + Admin API                                                        
   
  Per PRD milestone M3: "私有 AI（用户绑定 + 权限校验）→ 私有 AI 功能完整"                         
                                                            
  The AI core (detector, dispatcher, queue, providers, agent_api) is done in M2. What's missing is
  the user-facing CRUD for agents and the admin routes. All the query functions already exist in
  queries.js — M3 is purely about wiring up the HTTP routes.

  Step 1: Create src/api/agents.js — User Agent Management

  New file. 6 routes, all behind verifyJWT:

  ┌──────────────────────────────┬─────────────┬──────────────────────────────────────────────┐
  │            Route             │ Description │                  Key logic                   │
  ├──────────────────────────────┼─────────────┼──────────────────────────────────────────────┤
  │ GET /api/agents/mine         │ List my     │ listAgentsByOwner(request.user.id)           │
  │                              │ agents      │                                              │
  ├──────────────────────────────┼─────────────┼──────────────────────────────────────────────┤
  │                              │ Create      │ Generate rawToken + tokenHash +              │
  │ POST /api/agents             │ private     │ webhookSecret, createAgent(...), return      │
  │                              │ agent       │ rawToken & webhookSecret once                │
  ├──────────────────────────────┼─────────────┼──────────────────────────────────────────────┤
  │                              │             │ Ownership check via findAgentOwner, then     │
  │ PATCH /api/agents/:id        │ Update      │ updateAgent(id, fields). If api_key          │
  │                              │ config      │ provided, encrypt it. If webhook_url         │
  │                              │             │ cleared, also clear webhook_secret           │
  ├──────────────────────────────┼─────────────┼──────────────────────────────────────────────┤
  │ DELETE /api/agents/:id       │ Soft delete │ Ownership check, softDeleteAgent(id) — never │
  │                              │             │  hard delete (decision #8)                   │
  ├──────────────────────────────┼─────────────┼──────────────────────────────────────────────┤
  │ POST                         │ Regenerate  │ Ownership check, generate new rawToken,      │
  │ /api/agents/:id/rotate-token │ token       │ updateAgentToken(id, sha256(newRaw)), return │
  │                              │             │  new rawToken once                           │
  ├──────────────────────────────┼─────────────┼──────────────────────────────────────────────┤
  │ GET /api/agents/:id/logs     │ View call   │ Ownership check, listAgentLogs(agentId,      │
  │                              │ logs        │ limit)                                       │
  └──────────────────────────────┴─────────────┴──────────────────────────────────────────────┘

  Key constraints:
  - Agent name validated with AGENT_NAME_PATTERN_JS from detector.js in Fastify schema
  - model_type enum: openai, anthropic, custom_webhook
  - rawToken and webhookSecret only returned on create and rotate — never stored in plaintext
  - All schemas have additionalProperties: false

  Dependencies: None (queries already exist).

  Step 3: Add updateUserStatus query to src/db/queries.js

  Small addition — the only new query M3 needs:

  export function updateUserStatus(id, isActive) {
    const db = getDb()
    db.prepare('UPDATE users SET is_active = ?, updated_at = unixepoch() WHERE id = ?')
      .run(isActive ? 1 : 0, id)
    return findUserById(id)
  }

  Also block banning the system user (id=1).

  Dependencies: None.
